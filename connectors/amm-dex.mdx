AMM DEX connectors enable interaction with Automated Market Maker protocols through the Hummingbot Gateway middleware. These connectors provide access to decentralized liquidity pools, DEX aggregators, and concentrated liquidity protocols across multiple blockchain networks.

## How AMM DEX Connectors Work

AMM DEX connectors operate through the Gateway middleware, which provides standardized interfaces to different DeFi protocols:

### AMM Sub-Types

<Tabs>
  <Tab title="Router (DEX Aggregators)">
    Smart routing protocols that find optimal trade paths:
    
    **How they work:**
    - Split trades across multiple DEXs and pools
    - Optimize for best price execution
    - Minimize slippage and price impact
    - Compare routes in real-time
    
    **Examples**: 0x Protocol, Jupiter, Uniswap Auto Router
    
    **Best for**: Large trades, best price execution, multi-pool arbitrage
  </Tab>
  
  <Tab title="AMM (Constant Product)">
    Traditional automated market makers using x*y=k formula:
    
    **How they work:**
    - Liquidity providers deposit equal value of two tokens
    - Price determined by ratio in pool
    - Trades directly impact price (slippage)
    - Earn fees from swaps (typically 0.3%)
    
    **Examples**: Uniswap V2, SushiSwap, PancakeSwap
    
    **Best for**: Standard swaps, liquidity provision, simple arbitrage
  </Tab>
  
  <Tab title="CLMM (Concentrated Liquidity)">
    Advanced AMMs with customizable price ranges:
    
    **How they work:**
    - Liquidity providers set custom price ranges
    - Higher capital efficiency than traditional AMMs
    - Multiple fee tiers (0.01%, 0.05%, 0.3%, 1%)
    - Active liquidity management required
    
    **Examples**: Uniswap V3, Raydium CLMM, Meteora
    
    **Best for**: Advanced liquidity provision, concentrated market making
  </Tab>
</Tabs>

### Installing Gateway

Gateway is required for all AMM DEX connectors:

<Steps>
  <Step title="Install Gateway during setup">
    When setting up Hummingbot, answer `y` when prompted:
    ```bash
    make setup
    # Answer 'y' when prompted:
    Include Gateway? [y/N] y
    ```
  </Step>
  
  <Step title="Configure Gateway certificates">
    For production use with HTTPS:
    ```bash
    # In Hummingbot
    gateway generate-certs
    ```
    
    <Info>
    Gateway runs in development mode (HTTP) by default. Use `DEV=false` for production HTTPS mode.
    </Info>
  </Step>
  
  <Step title="Connect wallet to Gateway">
    Configure your wallet for the desired blockchain:
    ```bash
    gateway connect [chain]
    ```
    Enter your private key when prompted.
  </Step>
  
  <Step title="Test Gateway connection">
    Verify Gateway is running:
    ```bash
    gateway status
    ```
  </Step>
</Steps>

<Warning>
Gateway requires your wallet private key to sign transactions. Use a dedicated trading wallet, never your primary holdings wallet.
</Warning>

## Trading on AMM DEXs

### Understanding Slippage

AMMs experience price impact from trades:

<Info>
**Slippage Factors**
- Pool liquidity depth
- Trade size relative to pool
- Price volatility
- Network congestion (gas prices)

Larger trades = higher slippage
</Info>

<CodeGroup>

```python calculate_slippage.py
# Example: Price impact calculation
pool_liquidity = 1000000  # $1M pool
trade_size = 50000  # $50k trade

# Simplified price impact
impact_percentage = (trade_size / pool_liquidity) * 100
print(f"Estimated price impact: {impact_percentage}%")  # ~5%

# Actual impact depends on AMM curve
```

</CodeGroup>

<Tip>
**Minimizing Slippage**

1. Use DEX aggregators for large trades
2. Split large orders across multiple transactions
3. Trade during high liquidity periods
4. Choose pools with deeper liquidity
5. Use CLMM pools for better execution
</Tip>

## Liquidity Provision

AMM and CLMM connectors support liquidity provision:

<Tabs>
  <Tab title="Traditional AMM">
    Provide liquidity to constant product pools:
    
    **Process:**
    1. Deposit equal value of both tokens
    2. Receive LP tokens representing your share
    3. Earn fees from all swaps (pro-rata)
    4. Withdraw anytime by burning LP tokens
    
    **Risks:**
    - Impermanent loss from price divergence
    - Smart contract risk
    - Concentration risk in single pool
  </Tab>
  
  <Tab title="Concentrated Liquidity">
    Provide liquidity within custom price ranges:
    
    **Process:**
    1. Choose price range for your liquidity
    2. Deposit tokens (ratio depends on range)
    3. Earn fees when price is in range
    4. Rebalance as price moves
    
    **Advantages:**
    - Higher capital efficiency (up to 4000x)
    - Earn more fees per dollar
    - Customizable risk/return profile
    
    **Challenges:**
    - Requires active management
    - Higher impermanent loss if price exits range
    - More complex than traditional AMM
  </Tab>
</Tabs>

## OrderType

Enum defining the types of orders supported by Hummingbot.

**Source:** `hummingbot/core/data_type/common.py:8`

<ResponseField name="MARKET" type="int" default="1">
  Market order - executed immediately at the current market price
</ResponseField>

<ResponseField name="LIMIT" type="int" default="2">
  Limit order - executed only at the specified price or better
</ResponseField>

<ResponseField name="LIMIT_MAKER" type="int" default="3">
  Limit maker order - ensures the order is posted to the order book and not immediately filled
</ResponseField>

<ResponseField name="AMM_SWAP" type="int" default="4">
  AMM swap order - swap tokens through an automated market maker
</ResponseField>

<ResponseField name="AMM_ADD" type="int" default="5">
  Add liquidity to AMM/CLMM pool
</ResponseField>

<ResponseField name="AMM_REMOVE" type="int" default="6">
  Remove liquidity from AMM/CLMM pool
</ResponseField>

### Methods

<ResponseField name="is_limit_type()" type="bool">
  Returns `True` if the order type is `LIMIT` or `LIMIT_MAKER`
</ResponseField>
